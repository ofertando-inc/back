Project: Community deals platform for Colombia - Backend MVP

You are building the backend of a web application that helps users discover, publish, vote, and report promotional offers. The first target market is Colombia. The app does not process payments and does not sell products directly. It acts as a discovery and trust layer between users and external merchants or local stores.

The backend must be designed for an MVP that is realistic, maintainable, and fast to implement by a solo developer in an RNCP project context.

==================================================
1. TECH STACK (FROZEN)
==================================================

- Framework: NestJS
- Language: TypeScript
- Database: PostgreSQL
- ORM: Prisma
- Authentication: JWT
- Validation: DTOs + server-side validation
- Architecture: modular monolith
- API style: REST JSON

==================================================
2. PRODUCT SCOPE
==================================================

This is a community-driven deals platform.

Main features of the MVP:
- user registration and login
- simple user management
- public offer listing and detail pages
- authenticated users can create and edit their own offers
- users can vote on offers
- users can report offers
- admins can review reported offers and disable/reactivate offers
- admins can disable/reactivate users
- basic moderation and logging
- no payments
- no comments
- no personalized alerts
- no advanced reputation system
- no scraping in the MVP

5. BUSINESS RULES
==================================================

Users
- A user can have status ACTIVE or DISABLED.
- A DISABLED user cannot log in.
- A DISABLED user cannot create offers, vote, or report.

Offers
- Offer statuses are:
  - ACTIVE
  - REPORTED
  - DISABLED
  - DELETED

Offer permissions
- Authenticated users can edit only their own offers.
- Admins can edit any offer.
- A DELETED offer cannot be edited.
- A DISABLED offer can only be reactivated by an admin.

Expiration
- When endDate is in the past, the offer becomes DISABLED.
- Expired offers are not deleted.
- Expired offers are no longer publicly visible.

Votes
- One user can have only one vote per offer.
- If the user votes again on the same offer, the previous vote is replaced.
- Offer score = UP votes - DOWN votes.

Reports
- One user can submit only one report per offer.
- A report increments reportCount on the related offer.
- When reportCount reaches the defined threshold, the offer becomes REPORTED.
- Suggested threshold for MVP: 3 reports.

Deletion
- Offer deletion is soft deletion, not hard deletion.
- Soft deleted offers are not publicly visible.

Public visibility
- ACTIVE offers are publicly visible.
- REPORTED offers may remain visible with a warning if needed, but for the backend MVP you can keep public listing focused on ACTIVE offers first.
- DISABLED and DELETED offers are not publicly visible.

- title: minimum 5 characters
- description: minimum 20 characters
- storeName: required
- city: required
- offerType: required
- startDate: required
- endDate: required
- endDate must be greater than startDate
- if externalUrl is provided, it must be a valid URL

Auth validation:
- email must be valid
- password must meet a reasonable minimum length
- username required
